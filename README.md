# ゲームタイトル
「ゴース撮る」
## 概要
- プレイ人数　１人
- 対応機種　　PC
- ターゲット　写真を撮るのが好きな人
- 制作期間　　3ヶ月

このゲームは森を歩き回り、おばけの写真を撮影するゲームです。
おばけはこちらに気がつくと逃げてしまうので注意を払いながら撮影しなければいけません。
制限時間内に写真を10枚撮って高得点を目指しましょう。

## 使用技術
- C++
- DxLib
- HLSL（ポストエフェクト）
- FSM（有限状態マシン）
- Ray / Sphere / AABB 判定
- Visual Studio 2022

## 技術的な工夫
### 1. FSM（有限状態マシン）によるキャラクター制御
プレイヤーと敵キャラクターの行動を FSM で管理しています。  
状態（待機 / 巡回 / 発見 / 逃走 など）をクラスごとに分離し、  
遷移条件を明確化することで **拡張性と可読性の高い設計** を実現しました。

### 2.カメラに幽霊が写っているかの判定
カメラを覗いた際に、幽霊が視界に入っているかどうかを判定するため、  
**プレイヤー前方に“視界を表す三角形”を生成し、その内部に Sphere を敷き詰める方式** を採用しました。

#### ● 判定方法
1. プレイヤーの向きと視野角から視界三角形を設定  
2. 三角形内部に一定間隔で Sphere を配置  
3. 各 Sphere と幽霊の当たり判定（Sphere）をチェック  
4. いずれかが交差した場合 → **“写っている”** と判定  

#### ● この方式のメリット
- レイ1本より安定した視界判定が可能  
- 遮蔽判定と組み合わせることで、  
  **「物陰に隠れた幽霊は写っていない」** を実現 

### 3. 遮蔽判定（Occlusion）
幽霊とプレイヤーの間にオブジェクトがある場合、  
**AABB とレイの交差判定** を用いて遮蔽を検出。  
これにより、  
**「幽霊が物陰に隠れていると写らない」**  
というホラー演出を実現しています。

### 4. HLSL を用いたカメラ演出（Scanline + Grain Noise）
カメラを覗いた際の“映像越しの不気味さ”を表現するため、  
HLSL によるポストエフェクトを実装しました。

#### ● 実装したエフェクト
- 縞模様（Scanline）
- Grain ノイズ

#### ● 技術的アプローチ
- ピクセルシェーダで画面全体に縞模様を描画  
- 疑似乱数を用いて Grain ノイズを生成  

#### ● 演出意図
- “古いビデオカメラ越しの視界”を再現  
- 実際にカメラを覗いているような臨場感を演出


- パフォーマンスを考慮した軽量化
